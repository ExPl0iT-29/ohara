# Product Specification

## Primary User

Single user only.

No multi-user support.

No teams.

No collaboration.

The system is optimized entirely around one person's learning workflow.

---

## Supported Content Types

* Blogs
* Websites
* Documentation
* PDFs
* Research Papers
* YouTube Videos
* GitHub Repositories
* Books
* Tweets/X Threads
* Reddit Posts
* Any publicly accessible URL

Internally all of these should be normalized into a unified Content model.

---

## Core User Journey

Discover Content

↓

One-click Save

↓

Background Extraction

↓

Beautiful Reading Queue

↓

Read

↓

Mark Complete

↓

Search & Revisit

---

## Home Screen

The home screen is a reading dashboard.

Possible sections:

* Continue Reading
* Recently Saved
* Quick Reads
* Deep Reads
* Videos
* Papers
* Recently Finished
* Recommended for Today

No project-first navigation.

No folder-first navigation.

---

## Reader Experience

Every content item should provide:

* Hero Image
* Clean Reader View
* Estimated Reading Time
* Summary
* Metadata
* Reading Progress
* Offline Availability (where possible)

Reading should always feel better than opening the original webpage.

---

## Search

Search should work across:

* title
* extracted text
* summaries
* topics
* authors
* URLs

Search is intended to act as memory rather than keyword lookup.

---

## AI Features

Automatic:

* metadata extraction
* summaries
* topic extraction
* related content
* duplicate detection
* reading time estimation

Future:

* recommendation engine
* semantic search
* contextual resurfacing
* intelligent reading queue

---

## Projects

Projects are optional contextual relationships.

Projects never own content.

Content can belong to zero, one, or many projects.

Projects exist to provide context during work, not to organize the library.

---

## Non-Goals

Do NOT build:

* folder-heavy organization
* manual tagging workflows
* complex knowledge graphs
* social features
* collaboration
* gamification
* task management
* project management

The application is a reading companion, not a productivity suite.
